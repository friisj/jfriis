# Bonkhold - Definitions

> Glossary of terms specific to this project. Maintain as concepts evolve.

---

## Core Terms

| Term | Definition | Example |
|------|-----------|---------|
| Norse layer | Temporary, player-built visual elements — construction, tools, fortifications | Wooden palisades, repair scaffolding, storage racks |
| Roman layer | Permanent, hand-authored infrastructure that persists across wipes | Stone keeps, paved roads, arched tunnels, defensive walls |
| Deep layer | Endgame underground spaces with the strongest loot and environmental mystery | Catacombs, ancient vaults, subterranean chambers |
| Readability | Visual clarity of game elements at gameplay zoom — players can instantly parse what things are | Distinguishing a claimable fort from a decorative ruin at top-down scale |
| Authored places | Hand-designed locations with inherent strategic value, as opposed to player-spammed structures | A hilltop keep controlling access to a dungeon entrance |
| Asset generation pipeline | End-to-end process from concept/prompt to game-ready asset | Text prompt → image generation → cleanup → tiling → integration |

## Related Concepts

| Concept | Relationship to This Project |
|---------|------------------------------|
| HD-2D | Visual reference style (Adventures of Elliot) — high-detail 2D art with depth |
| Tilesets | Primary asset format for top-down games — generation must produce tileable output |
| Style transfer | Technique for applying consistent art direction across generated assets |
| Top-down perspective | Camera angle that defines all asset requirements — everything must read from above |
